Additional Public Information You Can Gather (All Free or Low-Cost)
Beyond your three pillars, here’s what’s reliably public and automatable:

Property fundamentals (from county appraisal districts / CADs): Year built, # of stories/units, construction type (frame, masonry, etc.), improvement value, land use code (commercial vs multi-family), sometimes roof material or # of layers.
Ownership & transaction history: Mailing address of owner (often an LLC), recent sales/deeds, tax delinquencies/liens (flags motivated sellers), exemptions.
Management company / registered agent: For LLC-owned commercial buildings, state Secretary of State (SOS) business filings give registered agent name/address/phone (public, searchable). Many counties list property manager on commercial records.
Code enforcement / violations: Open records on building violations, roof-related complaints, or failed inspections.
Zoning / hazard overlays: Flood zones, wind/hail zones (FEMA + local GIS), which amplify hail risk scoring.
Historical changes: Public aerial imagery archives (county GIS or Google Earth historical) to visually confirm roof age/replacement (or feed into a cheap vision model).
Related activity: Solar/HVAC/structural permits on the same building (signals overall maintenance budget).

TX-specific goldmine (since you’re in Dallas):

DCAD / Tarrant / Collin / Denton CADs offer free bulk CSV + GIS downloads with sq ft, owner, year built, etc.
TxGIO statewide parcel layer (free standardized GIS from appraisal districts).
Dallas OpenData / Socrata APIs for permits & violations.
Harris County (Houston) example shows similar open bulk downloads.

Nationwide you can layer in free NOAA bulk data or open county portals; for scale, paid aggregators (ATTOM, BatchData) are basically repackaged public records but normalized.

How I Would Build the System (Behind-the-Scenes Agent Architecture)
I’d make this a SaaS platform with autonomous AI agents that run 24/7, not a one-off script. Contractors log in, pick their service radius, and get a live dashboard of scored leads with one-click export (CSV, CRM push, or direct-mail list).
Core Tech Stack (MVP in 8–12 weeks for one dev + you):

Orchestration: CrewAI or LangGraph (multi-agent framework). Each agent has specific tools + memory.
Data Layer: PostgreSQL + PostGIS (for spatial hail matching). Redis for queues.
Ingestion: Apache Airflow or Prefect for scheduled jobs + event triggers.
Agents (run in parallel behind the scenes):
Geo-Filter Agent: Loads parcels ≥2,500 sq ft from CAD GIS/CSV in target counties.
Permit Agent: Queries county permit portals (Dallas uses Accela/Socrata – API-friendly) or ATTOM bulk for “roof replacement” keywords + date.
Hail Agent: Downloads/processes NOAA Storm Events CSV (free, updated monthly) or hooks Xweather/HailStrike API for real-time alerts. Does spatial join (building lat/long vs hail reports).
Owner/Mgmt Researcher Agent:
Pulls owner from CAD.
If LLC → hits TX SOS public search (or OpenCorporates API) for registered agent/contact.
Google/Bing search “[address] property management” + LinkedIn-style enrichment (ethical, public only).

Scorer & Enricher Agent: Calculates score, generates one-page PDF profile (roof age, hail hits, owner details, map).
Monitor Agent: Daily/after-storm trigger — scans new permits or hail events and pushes alerts.

Frontend: Streamlit or Next.js dashboard with Leaflet map (pins colored by score), filters, export.
Automation triggers:
Daily cron for new permits.
Webhook or poll after major hail (NOAA or paid alert service).
User-requested full-market scan.
